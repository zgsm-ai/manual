# Plan 模式

Plan 模式帮助你将复杂任务分解为可管理的步骤。

## 概述

面对复杂的开发任务时，Plan 模式会在执行前创建结构化的实施计划。

## 使用方法

### 创建计划

```bash
costrict plan "添加用户认证系统"
```

### 执行计划

```bash
costrict execute-plan plan-2024-02-02.md
```

### 交互式规划

```bash
costrict chat
> 我需要重构数据库层
> 你能为此创建一个计划吗？
```

## 计划结构

计划通常包括：

- 任务分解
- 需要修改的文件
- 测试策略
- 实施顺序
- 回滚考虑

## 最佳实践

- 对需要多个文件的任务使用 plan 模式
- 执行前审查计划
- 将大型计划分解为更小的里程碑
- 保持计划专注于单一功能
